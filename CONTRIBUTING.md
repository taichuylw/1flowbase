# Contributing to 1flowbase

Thanks for your interest in contributing to 1flowbase.

## Contributor License Agreement

Before a pull request can be merged, each contributor must accept the
1flowbase Individual Contributor License Agreement in [CLA.md](CLA.md).

The repository uses CLA Assistant as the signing gate. When you open a pull
request, CLA Assistant will comment with a link to review and accept the CLA if
you have not already signed it. After acceptance, the CLA status check should
pass automatically.

Do not submit a contribution if you are not allowed to grant the rights described
in the CLA. If you are contributing work owned by an employer or another
organization, make sure you have permission before opening a pull request.

## Pull Requests

- Keep pull requests focused on one change.
- Follow the local instructions in `AGENTS.md` and any nested `AGENTS.md` files
  for the area you are editing.
- Include tests or verification evidence when the change affects behavior.
- Do not include secrets, credentials, private keys, customer data, or other
  confidential material.

## Maintainer Setup

Maintainers should configure CLA Assistant for this repository and point it to
the published version of `CLA.md`, such as the file URL on the default branch or
a dedicated Gist.

After CLA Assistant is enabled, protect the target branch and require the CLA
Assistant status check before pull requests can be merged.
