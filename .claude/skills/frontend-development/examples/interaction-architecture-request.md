# Interaction Architecture Request Example

```md
## 任务理解

我理解你要改的是一个双栏设置类页面，
重点不是单纯换布局，而是把“看什么、点哪里、进入哪层详情、从哪里做管理和执行”这套交互规则收稳。

## 交互架构快审

- 首屏主任务：用户进入页面后先定位对象，再决定是查看详情还是进入管理
- L0 概览：左侧导航 + 右侧摘要，不在首屏塞完整管理表单或 AI 执行
- L1 聚焦：单条对象详情统一走 `Drawer`
- L2 管理：完整列表、搜索、筛选、批量操作进入独立管理页
- L3 执行：复杂执行或 AI 协作通过明确动作按钮进入专用执行流
- 一致性规则：同类对象统一点击结果，不能一部分开 `Drawer`，另一部分直接跳页

## 设计需求草案

- 页面目标：让用户快速定位对象，稳定地查看详情并进入管理 / 执行
- 主要对象：设置项 / 资源对象
- 页面结构：左侧导航，右侧内容区，摘要区只承接当前 section 的核心信息和主入口
- 关键模块：section 导航、对象列表 / 摘要、状态、主操作、空态 / 错误态
- 页面交互：先浏览或筛选对象，再通过统一详情入口查看单条信息；需要全量操作时进入 L2 页面；复杂执行从明确按钮进入 L3
- 关键状态：loading、empty、error、success、running、selected
- 视觉约束：沿用当前项目浅底控制台语法，不引入新的产品级交互语言

## 明确建议

我建议先把 L1 / L2 / L3 规则固定，再决定双栏里每个区块怎么摆。
这样能避免页面做着做着同时混进导航、详情、管理和执行，最后只剩一张好看的拼盘。
```
