# Vague Page Request Example

```md
## 任务理解

我理解你要做的是一个设置类管理页面，
它的目标是让用户围绕【主对象】完成查看、筛选、配置和执行动作。

## 参考拆解

这次没有明确参考图，我会优先沿用当前项目现有 shell recipe 和 `DESIGN.md`。

## 交互架构快审

- 首屏主任务：先定位对象，再完成核心管理动作
- L1 聚焦：如需单条详情，统一进入 `Drawer / Inspector`
- L2 管理：全量筛选、搜索、批量操作进入独立管理页
- L3 执行：复杂执行或 AI 协作通过明确动作进入专用流程
- 一致性规则：同类对象不混用多种点击结果

## 设计需求草案

- 页面目标：让用户在一个稳定的工作台页面内完成核心管理动作
- 主要对象：【待实现对象】
- 核心动作：【动作 1】、【动作 2】
- 页面结构：顶部标题与工具条，中部主列表 / 主卡片区，必要时保留辅助信息区
- 关键模块：筛选、列表 / 卡片、状态、主操作、空态 / 错误态
- 页面交互：用户先通过筛选定位对象，再在主列表 / 主卡片区触发主操作；操作反馈优先在当前页内联更新，必要时再进入 `Drawer / Inspector`
- 关键状态：loading、empty、error、success，以及当前对象需要的运行态
- 视觉约束：沿用当前项目浅底工作台风格，不引入新的产品级视觉母本

## 明确建议

我建议先按当前项目既有 recipe 做一个可运行版本，把对象、动作和状态先落地，
避免在需求还模糊时直接发散成新的页面语法。
如果过程中出现阻塞性产品分歧，我会一次性列出来。
```
