# Frontend Requirement Brief Template

以下模板用于直接回复用户 / 客户，不是内部草稿。

## 完整版

```md
## 任务理解

我理解你要做的是一个【页面名称 / 页面类型】，
它的目标是【解决的问题】，
主要服务【目标用户】。

## 参考拆解

这次参考图 / 外部样本，我建议只借：
- 【布局骨架】
- 【信息层级】
- 【组件组合 / 页面节奏】

这次不直接照搬：
- 【品牌色 / 文案 / 图标 / 第三方产品皮肤】
- 【与 DESIGN.md 冲突的状态表达】

## 设计需求草案

- 页面目标：【一句话】
- 主要对象：【对象】
- 核心动作：【动作 1】、【动作 2】
- 页面结构：【顶部 / 侧栏 / 主区 / 辅助区】
- 关键模块：【模块列表】
- 页面交互：【主路径 / 关键反馈 / 模块协作方式】
- 关键状态：【loading / empty / error / success / running / selected / draft 中需要的项】
- 视觉约束：【必须遵守的 DESIGN.md 规则】

## 明确建议

我建议按【推荐方向】推进，
原因是【为什么这样最符合当前项目和参考图的共同约束】。
接下来我会先按这版需求直接实现；
如果过程中遇到阻塞性产品分歧，我会一次性列出来。
```
